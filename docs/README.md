## Documentation policy

1. Keep docs aligned with actual implementation (no aspirational claims without labels).
2. Clearly separate **implemented** vs **planned** capabilities.
3. Include validation commands for each operational procedure.
4. Update docs in the same PR/commit as feature changes.
